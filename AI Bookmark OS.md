项目目标：基于以下两个 GitHub 开源项目源码，合并重构并开发一个功能完整、体验一致、UI 统一的 AI 书签管理项目：

- bookmark-pilot：https://github.com/BOOHHP/bookmark-pilot
- Markline：https://github.com/jdf12/Markline

请完成以下任务：

1. 源码获取与分析
   - 下载并阅读 bookmark-pilot 与 Markline 的完整源码。
   - 分析两个项目的技术栈、目录结构、核心模块、数据模型、前后端交互方式、依赖库、构建方式和运行方式。
   - 梳理两个项目分别提供的核心功能，并明确哪些功能需要完整保留、哪些功能可以合并、哪些功能需要重构优化。
   - 输出一份功能对照表，包含：
     - bookmark-pilot 原有核心功能
     - Markline 原有核心功能
     - 合并后新项目中的对应实现方式
     - 是否保留、增强、合并或重构

2. 项目合并与架构整合
   - 以现有源码为基础进行改造，不从零重写。
   - 设计一个统一的新项目架构，整合两个项目中重复或冲突的模块。
   - 统一前端路由、状态管理、API 调用、数据存储、配置文件、构建流程和部署方式。
   - 保留两个原项目的全部核心能力，并消除重复实现、无效代码和不一致的交互逻辑。
   - 对关键模块进行合理分层，包括但不限于：
     - 书签采集与导入模块
     - 书签分类与标签模块
     - AI 分析与内容理解模块
     - 搜索与筛选模块
     - 用户界面与交互模块
     - 数据持久化与同步模块
     - 配置与扩展模块

3. 功能完整性要求
   - 合并后的项目必须完整覆盖 bookmark-pilot 与 Markline 的全部核心功能。
   - 至少包含以下能力：
     - 新增、编辑、删除、收藏、归档书签
     - 书签标题、链接、描述、标签、分类、备注等信息管理
     - 书签导入与导出
     - 多维度搜索、过滤、排序
     - 标签体系与分类体系
     - AI 自动摘要、AI 标签推荐、AI 分类建议或内容理解能力
     - 书签内容预览或元信息抓取
     - 批量操作
     - 用户配置与偏好设置
     - 响应式布局，适配桌面端和移动端
   - 如果两个原项目中存在功能差异，应优先采用功能更完整、结构更清晰、扩展性更好的实现方案。

4. 功能深化与体验优化
   - 在保留原有能力的基础上，对核心流程进行体验优化。
   - 优化书签添加流程，使用户能够快速完成链接保存、自动识别标题、生成摘要、推荐标签和分类。
   - 优化搜索体验，支持关键词、标签、分类、时间、状态等组合筛选。
   - 优化 AI 功能调用流程，提供加载状态、失败提示、重试机制和结果可编辑能力。
   - 优化空状态、错误状态、加载状态和批量操作反馈。
   - 优化项目整体性能，避免不必要的重复请求、重复渲染和大型列表卡顿。

5. Apple OS 风格 UI 统一

     - 清晰的信息层级
     - 合理留白
     - 平滑动效
     - 类 macOS / iOS 的侧边栏、工具栏、弹窗、按钮、输入框和列表样式
   - 统一颜色、字体、间距、圆角、阴影、图标、交互动效和组件规范。
   - 建立统一的设计系统或基础组件库，包括：
     - Button
     - Input
     - Select
     - Modal/Dialog
     - Sidebar
     - Toolbar
     - Card
     - Tag
     - Empty State
     - Loading State
     - Toast/Notification
   - 确保不同页面之间视觉风格一致，不保留两个原项目割裂的 UI 风格。

6. 代码质量与可维护性
   - 清理无用代码、重复依赖、废弃配置和无效资源。
   - 统一命名规范、目录规范、代码风格和注释风格。
