# Apple Dynamic Glassmorphism UI 说明

AI Bookmark OS 界面统一采用 Apple 风格的动态玻璃拟态（Dynamic Glassmorphism）视觉语言，并参考 `apple-design` skill 的响应、材料、动效与无障碍原则：简洁、通透、精致、未来感，同时保持业务信息结构不变。

## 设计原则

- 银白 / 冰蓝 / 浅灰 / 冷白为主，低饱和冷色渐变背景。
- 多层毛玻璃面板 + 细边框 + 柔和阴影 + 内高光，避免厚重装饰。
- 圆角统一为大圆角体系（约 12 / 18 / 24）。
- 主按钮使用冰蓝微光渐变；次按钮为半透明描边玻璃。
- 动效克制：100–380ms，按压反馈即时（pointer-down），支持 `prefers-reduced-motion`。
- 材料可降级：`prefers-reduced-transparency` / `prefers-contrast`。
- 文本与玻璃背景保持足够对比，优先可读性。

## 样式来源

- `src/styles/apple-design-system.css`：统一设计令牌与玻璃组件语言（同步至 `src/timeline/shared/`）
- `src/timeline/shared/apple-page-polish.css`：Popup / Settings / Standalone / Graph / Checker 统一抛光与实色中和
- `src/sidepanel/sidepanel.css`：AI 分类侧边栏玻璃壳层
- `src/bookmark-nav/bookmark-nav.css`：书签导航 sticky 顶栏搜索、卡片、悬浮操作、页脚

## 统一设计变量

- 颜色令牌：背景、表层、分隔线、正文、次要文字、危险状态和强调色全部从共享 CSS 变量取值；页面不得为同一语义再定义一套近似色。
- 字体令牌：`--app-font` 和 `--app-font-size` 由设置页驱动，正文以系统字体栈为回退；数字、URL 与长标题需保持可读且允许换行。
- 间距与尺寸：以 4px 为基本步进；工具栏、输入框、图标按钮使用稳定高度，避免加载、悬停或动态文案引起布局跳动。
- 圆角与层级：控制在 8px、12px、18px、24px 的有限尺度内。卡片只用于重复条目、模态框和需要明确边界的工具，页面区段保持无卡片化的连续布局。
- 阴影与边框：玻璃材质使用低对比边框与短阴影；高对比或减少透明度模式下退化为不透明表层，不能依赖模糊效果传达边界。

## 组件规范

- 顶栏与导航：提供清晰页面标题、主操作与可见焦点；在长页面使用 sticky 行为时，不得遮挡锚点、搜索结果或弹窗。
- 按钮：图标按钮只承载熟悉的单一操作，并提供 `title` 与可访问名称；有文字的按钮仅用于明确命令。危险操作使用确认步骤。
- 输入与搜索：始终保留可见标签或等价的 `aria-label`；搜索结果、空状态、错误和加载状态在原上下文呈现，不以颜色作为唯一信号。
- 树与列表：分类树的展开、重命名、删除和拖拽均保持键盘可达；长名称截断时仍能通过标题或焦点态获得完整内容。
- 模态框：打开后焦点进入对话框，关闭后回到触发控件；Esc 可关闭非破坏性对话框，确认与取消操作的视觉主次明确。
- 进度与通知：AI 请求、重试、取消和书签应用过程显示可理解的阶段与数量；错误说明可操作，不泄露 API 密钥或原始请求内容。

## 页面覆盖

- AI 分类 Side Panel：悬浮玻璃顶栏、搜索条、操作条、进度卡、弹窗
- 书签导航页：固定悬浮顶栏 + 搜索、模块化卡片网格、悬浮操作区、分层页脚
- Popup / 完整窗口 / 设置 / 图谱 / 健康检查：共享 glass 顶栏、卡片、输入与主按钮

## 交互与可用性

- 按钮/图标按压 `scale(0.97)` 即时反馈
- 卡片/列表 hover 轻微上浮与边框高亮
- sticky 导航/搜索在滚动时保持可用
- reduced-motion / reduced-transparency / higher-contrast 可降级

## 响应式与可用性

- 支持侧边栏窄宽度、平板和桌面窗口；窄屏优先保留搜索、主操作和当前任务进度，次要操作可折叠但不应消失。
- 文本不随视口宽度缩放；使用弹性容器、最小/最大宽度与换行约束，确保长书签标题、URL、翻译文案不会覆盖相邻控件。
- 交互目标在触摸环境中保持足够可点击面积，拖放操作应有非拖放替代路径，例如重命名、删除与可见的移动目标。
- 所有状态同时满足鼠标、键盘与系统辅助设置。焦点指示必须在玻璃背景上清晰可见，不能只依赖 hover。
- 浅色、深色与系统模式共用同一语义色体系；任何新增页面至少检查这三种模式中的文本对比和边界可见性。

## 验收

- `npm run typecheck`：检查 TypeScript 与 Chrome API 类型。
- `npm run build`：生成 AI 界面和可加载的 `dist/` 扩展。
- `npm run test:quick-bookmark`：覆盖快速收藏建议和重复书签处理的回归路径。
- `npm run preview:check`：验证构建产物入口与关键资源。
- `npm run audit:project`：验证交付文件、统一入口、AI 重试设置与文档要求。
- 人工检查：在侧边栏、Popup、设置、书签导航、图谱和健康检查页分别以窄宽度与桌面宽度确认无溢出、焦点可见、文案完整；再检查深色、浅色和减少动态效果模式。
