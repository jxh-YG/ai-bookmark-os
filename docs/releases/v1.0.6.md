# AI Bookmark OS v1.0.6

本版本集中修复 AI 辅助分类与金字塔分类的测试连接链路，完善 AI 请求的代理、重试和流式解析，并补齐图谱、链接检查、数据校验与端到端验证。

## AI 测试连接

- AI 辅助分类和金字塔分类在用户点击“测试连接”时，按当前 API 服务商主机申请可选站点权限；不再依赖链接检查功能预先获得 `<all_urls>` 权限。
- 自定义 OpenAI、Anthropic 和 Gemini 服务商均按所选协议构造请求。Anthropic Base URL 填写到 `/v1` 时不再重复拼接；Gemini Base URL 会正确补充模型生成路径。
- “使用完整 URL”开关会保留用户输入的完整请求地址，不会再截断 OpenAI、Anthropic 或 Gemini 的路径后重复拼接。
- 金字塔测试连接统一通过 Service Worker 代理发送，并按配置执行超时和重试；遇到 408、429、5xx 或可恢复网络错误会按指数退避重试。
- 测试成功不仅检查 HTTP 2xx，还会验证 OpenAI、Anthropic、Gemini 响应结构确实包含文本回复；供应商返回的 JSON 错误和无效响应会明确显示为失败。
- 移除已过期的 Anthropic beta 请求头，保留标准 `anthropic-version` 协议头。

## AI 分类可靠性

- AI 分类请求通过 Service Worker 代理，避免扩展页面跨域预检造成的延迟和失败；SSE 流式响应会汇总为完整文本后再解析。
- 补齐分类失败后的队列释放、终态记录、进度回显、单书签补偿重试和批次恢复，避免失败任务长期卡住。
- 强化导入的分类结果与标签缓存校验，防止无效数据写入本地存储。

## 使用体验与数据一致性

- 图谱入口重新打开已有图谱标签页时会刷新当前扩展包，避免继续使用过期脚本；滚轮缩放的端到端范围已覆盖。
- 链接检查修复重复点击竞态、响应有效性识别和后台消息所有权处理；可达链接会正确显示为正常。
- 统一标签颜色读取与页面路由行为，修复目录移动、删除后的镜像同步及若干通知、订阅和页面导航边界问题。

## 验证

## 安装

下载 `ai-bookmark-os-v1.0.6-chrome-extension.zip`，完整解压后，在 Chrome 的 `chrome://extensions` 页面开启开发者模式，选择解压后直接包含 `manifest.json` 的目录加载。
